Debugging Errors

In the windows port there are two executables: deskshell.exe and deskshell_debug.exe If you run the second executable then it will open a console window that will show debug messages and errors from the backend.

To launch an application with the debug executable you can rename it from app.desk to app.desk-debug. Clicking on the app will then automatically launch it will the debug console window.

If you are developing the source code of deskshell then you may run into startup errors that prevent the application from being able to be launched. In this case it can be difficult to know what is going wrong. It is possible to create a .bat file and use that to directly launch the deskshell code and get any fatal errors. An example bat file is shown below. Set DIR variable to point to the deskshell directory and then change app.desk to be the name of your desk file. If you rename the file to be .desk-debug you will also get a console window open. 

rem set DIR to hold path to Deskshell folder.
set DIR=%cd%\..\..\Deskshell
%DIR%\bin\win\node.exe %DIR%\bin\win\deskshell.js %cd%\app.desk
pause

Fixed Port

It is possible to run your application on a fixed port, this won't allow it have multiple instances, but it does make it easier to connect chrome and chrome debug tools or another browser / dev tools to the application port and then debug. Note that if you do this the application should function but the chrome remote debugging api calls will go to the deskshell window not your browser window.
